# 모바일 접속 문제 해결

## 현재 상태
- 서버는 이미 `0.0.0.0:3000`에서 리스닝 중 (외부 접속 허용)
- IP 주소: `192.168.219.41`

## 해결 방법

### 방법 1: 브라우저 개발자 도구 모바일 모드 (가장 쉬움) ⭐

1. **Chrome/Edge 브라우저 열기**
2. **F12** 또는 **Ctrl+Shift+I**로 개발자 도구 열기
3. **Ctrl+Shift+M** 또는 휴대폰 아이콘 클릭
4. **모바일 기기 선택:**
   - iPhone 12/13/14 Pro
   - Samsung Galaxy S20/S21
   - 또는 Custom: 375x812
5. **주소 입력:** `http://localhost:3000`
6. **직원 계정으로 로그인**

이 방법이 가장 빠르고 안정적입니다!

---

### 방법 2: 실제 모바일 기기 (방화벽 설정 필요)

#### 1단계: Windows 방화벽 설정

1. **제어판** → **Windows Defender 방화벽** → **고급 설정**
2. **인바운드 규칙** → **새 규칙**
3. **포트** 선택 → **다음**
4. **TCP** 선택 → **특정 로컬 포트: 3000** → **다음**
5. **연결 허용** → **다음**
6. **도메인, 개인, 공용** 모두 체크 → **다음**
7. **이름:** "Next.js Dev Server" → **마침**

#### 2단계: 개발 서버 재시작

현재 실행 중인 서버를 중지하고:

```bash
npm run dev:mobile
```

또는 PowerShell에서:
```powershell
npm run dev:mobile
```

#### 3단계: 모바일에서 접속

1. **모바일 기기가 같은 WiFi에 연결되어 있는지 확인**
2. **모바일 브라우저에서:**
   ```
   http://192.168.219.41:3000
   ```
3. **직원 계정으로 로그인**

---

### 방법 3: 방화벽 명령어로 빠르게 설정

PowerShell을 **관리자 권한**으로 실행하고:

```powershell
New-NetFirewallRule -DisplayName "Next.js Dev Server" -Direction Inbound -LocalPort 3000 -Protocol TCP -Action Allow
```

---

## 문제 해결

### "사이트에 연결할 수 없습니다" 오류

1. **방화벽 확인:**
   - Windows 방화벽에서 포트 3000 허용 확인

2. **IP 주소 확인:**
   ```bash
   ipconfig | findstr IPv4
   ```
   - IP가 변경되었을 수 있음

3. **네트워크 확인:**
   - 모바일과 PC가 같은 WiFi 네트워크인지 확인
   - 모바일 데이터가 아닌 WiFi를 사용해야 함

4. **서버 상태 확인:**
   ```bash
   netstat -ano | findstr :3000
   ```
   - `0.0.0.0:3000` 또는 `[::]:3000`이 보여야 함

### 권장: 브라우저 개발자 도구 사용

실제 모바일 기기보다 **브라우저 개발자 도구의 모바일 모드**를 사용하는 것이 더 빠르고 안정적입니다!
