# 모바일 앱 테스트 가이드

## 방법 1: 실제 모바일 기기에서 테스트

### 1단계: 개발 서버를 외부 접속 가능하도록 시작

```bash
npm run dev:mobile
```

또는 기존 서버를 중지하고 다시 시작:

```bash
# 기존 서버 중지 (Ctrl+C)
npm run dev:mobile
```

### 2단계: 모바일 기기에서 접속

1. **모바일 기기가 같은 WiFi 네트워크에 연결되어 있는지 확인**
2. **모바일 브라우저에서 접속:**
   - 주소: `http://192.168.219.41:3000`
   - 또는 컴퓨터의 실제 IP 주소 확인: `ipconfig` (Windows) 또는 `ifconfig` (Mac/Linux)

3. **로그인:**
   - 직원 계정으로 로그인
   - 로그인 후 자동으로 `/mobile-dashboard`로 이동

4. **PWA 설치 (선택사항):**
   - 모바일 브라우저에서 "홈 화면에 추가" 옵션 선택
   - 앱처럼 실행 가능

## 방법 2: 브라우저 개발자 도구 모바일 모드

### Chrome/Edge:

1. **F12** 또는 **Ctrl+Shift+I**로 개발자 도구 열기
2. **Toggle device toolbar** 버튼 클릭 (Ctrl+Shift+M)
3. **모바일 기기 선택:**
   - iPhone 12/13/14 Pro
   - Samsung Galaxy S20/S21
   - 또는 Custom 설정

4. **접속:**
   - `http://localhost:3000` 또는 `http://192.168.219.41:3000`

## 방법 3: 로컬 네트워크 테스트

### Windows 방화벽 설정:

1. **제어판** → **Windows Defender 방화벽** → **고급 설정**
2. **인바운드 규칙** → **새 규칙**
3. **포트** 선택 → **TCP** → **특정 로컬 포트: 3000**
4. **연결 허용** → **도메인, 개인, 공용** 모두 체크
5. **이름:** "Next.js Dev Server"

## 테스트 계정 생성

직원 계정이 없다면:

1. **관리자 계정으로 로그인** (`apeleba@naver.com` 또는 다른 관리자)
2. **직원 관리** 페이지로 이동
3. **새 직원 추가:**
   - 이메일: `staff@test.com`
   - 비밀번호: `test123456`
   - 이름: `테스트 직원`
   - 역할: `staff`
   - 매장 배정

4. **생성한 계정으로 로그인하여 테스트**

## 문제 해결

### 로그인이 안 되는 경우:

1. **Supabase 계정 확인:**
   - Supabase 대시보드 → Authentication → Users
   - 계정이 존재하는지 확인

2. **환경 변수 확인:**
   - `.env.local` 파일에 `NEXT_PUBLIC_SUPABASE_URL`과 `NEXT_PUBLIC_SUPABASE_ANON_KEY`가 올바른지 확인

3. **브라우저 콘솔 확인:**
   - F12 → Console 탭에서 에러 메시지 확인

### 모바일에서 접속이 안 되는 경우:

1. **방화벽 확인:**
   - Windows 방화벽에서 포트 3000 허용

2. **IP 주소 확인:**
   - `ipconfig`로 현재 IP 주소 확인
   - 변경되었을 수 있음

3. **네트워크 확인:**
   - 모바일과 PC가 같은 WiFi에 연결되어 있는지 확인
